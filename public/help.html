FactorySim Documentation

Complete reference for the FactorySim desktop digital-twin platform for manufacturing simulation.

Getting Started

What Is FactorySim?

FactorySim is a desktop application for building discrete-event simulation models of manufacturing systems. It combines a visual drag-and-drop factory builder with a SimPy-based simulation engine, letting you model production lines, job shops, assembly cells, and other manufacturing layouts without writing code.

Key capabilities:

  • Visual factory layout editor with 14 node types
  • Discrete-event simulation powered by SimPy 4.1 (Python 3.11+)
  • OEE, throughput, cycle time, WIP, and utilization KPIs
  • What-if scenario analysis with side-by-side comparison
  • Multi-product routing with per-product cycle time overrides
  • Export models as standalone Python/SimPy scripts
  • Excel/CSV import and export for batch parameter editing
  • Run artifacts (screenshots, KPIs, event logs) auto-saved per run

Your First Model

  1. Open the Factory Builder page from the sidebar.
  2. Drag a Source onto the canvas — this generates parts.
  3. Drag a Station — this processes parts. Set its cycle time.
  4. Drag a Sink — this collects finished parts.
  5. Connect them: click a node’s output handle and drag to the next node’s input handle.
  6. Go to Orders > Products to define at least one product with a routing through your station.
  7. Press Ctrl+S to save your model.

Running a Simulation

  1. Navigate to the Dashboard.
  2. Set the simulation duration (e.g., 8 hours for a shift).
  3. Click Run Simulation. Progress appears in the header bar.
  4. When complete, KPI cards and charts update automatically.
  5. Use the widget configurator to choose which charts to display.
  6. Download event logs or screenshots from the dashboard toolbar.
Tip: Use multiple replications (Settings > Simulation Defaults > Replications) with different random seeds to get statistically meaningful results. Confidence intervals are shown when replications > 1.

App Pages

Dashboard

The main results view after running a simulation. Displays KPI summary cards (OEE, throughput, cycle time, WIP), interactive charts (utilization bar chart, throughput-over-time, WIP trend, bottleneck heatmap, OEE breakdown), and a station data table. Use the widget configurator to toggle, reorder, and resize widgets.

The dashboard auto-caches a screenshot 2 seconds after results arrive. Run artifacts (screenshot, KPIs JSON, event log) are saved to the runs/ folder in your user data directory.

Factory Builder

A visual, drag-and-drop editor for constructing your factory model. The left panel lists all 14 available node types. Drag a node onto the canvas, then configure its parameters in the right-side property panel. Connect nodes by dragging from an output handle to an input handle.

Features:

  • Pan (click & drag on canvas) and zoom (Ctrl++ / -)
  • Select a node and press Delete to remove it
  • Click a connection to select it, then Delete to remove
  • Start from a template via the template selector button
  • Undo / Redo with Ctrl+Z / Ctrl+Shift+Z

Scenarios

Run what-if analyses by overriding model parameters without changing the base model.

  • Quick Scenarios — 10 pre-defined what-if templates (e.g., “Double buffer capacity”, “Add parallel station”, “Reduce cycle time by 20%”). One-click launch, results compared to baseline.
  • Custom Scenarios — pick any entity (station, buffer, source, etc.) and override specific parameters. Save the scenario for later re-use.
  • Comparison — select two or more scenarios to view side-by-side KPI differences with percentage changes.

Orders

Three sub-tabs:

  • Product Manager — define products, set arrival rates, priority, and routing sequences through your factory nodes.
  • Order Manager — create production orders specifying product, quantity, and due date. Sources in “orders” feed mode pull from this queue.
  • Delivery Predictions — view estimated completion dates based on the latest simulation results.

Code Editor

Exports your current factory model as a standalone Python script using SimPy. The generated code is fully runnable outside FactorySim — useful for advanced customization, CI/CD pipelines, or sharing with colleagues who don’t have FactorySim installed.

Click Export to generate, then Copy or Save the script.

Data Sync

Import model parameters from external files or configure live data connectors.

  • CSV Import — upload a CSV file with station/buffer parameters and map columns.
  • Excel Import — upload an Excel workbook. Supports sheet selection and column mapping.
  • Connectors (extensible) — placeholder for MES, MQTT, OPC-UA integration. Configure connector type and settings.

Settings

User profile, simulation defaults (duration, replications, seed), appearance (theme, language), data & storage management (export data, clear cache), and the role permissions matrix.

Node Types Reference

FactorySim provides 14 node types. Drag them from the left panel in the Factory Builder.

Station

The primary processing node. Parts arrive, are processed for a cycle time, and exit.

ParameterDescriptionDefault
Cycle TimeTime to process one part. Configurable distribution (constant, normal, exponential, triangular, uniform, lognormal, weibull).60s constant
Setup TimeOptional time for changeover between batches or products. Also a distribution.None
MTBFMean Time Between Failures (hours). Triggers random breakdowns.None (no failures)
MTTRMean Time To Repair (hours). Duration of each breakdown.None
Scrap RatePercentage of parts scrapped after processing (0–100%).0%
Batch SizeNumber of parts processed together before releasing.1
Shift ScheduleDefine shifts with start/end hours and active days. Station is off-shift outside these windows.24/7
Per-Product Cycle TimesOverride the default cycle time for specific products.Uses default

Buffer

A queue that holds parts between processing steps. Provides decoupling and absorbs variability.

ParameterDescriptionDefault
CapacityMaximum number of parts the buffer can hold.10
Queue RuleFIFO (first-in-first-out), LIFO (last-in-first-out), or PRIORITY (by product priority).FIFO

Source

Generates parts and feeds them into the factory. Every model needs at least one source.

ParameterDescriptionDefault
Feed Modeinterval — generates parts at a fixed rate. orders — pulls from the order queue.interval
Arrival RateSeconds between consecutive arrivals (interval mode only).60s
Product FilterOptional: restrict this source to spawn only a specific product type.All products
Product Batch SizeConsecutive parts of the same product type before switching to the next. 1 = round-robin.1

Sink

Exit point for finished parts. Records throughput and end-to-end cycle time. No configurable parameters.

Conveyor

Transports parts between nodes with travel time based on length and speed.

ParameterDescriptionDefault
LengthConveyor length in meters.10 m
SpeedTravel speed in meters per second.1 m/s
CapacityMaximum number of items on the conveyor simultaneously.10

Inspection

Quality check station. Parts are inspected; those that fail are routed out as defects.

ParameterDescriptionDefault
Inspection TimeSeconds per inspection.30s
Defect RatePercentage of parts found defective (0–100%).5%
Inspection Typevisual, automated, or sampling.visual

Operator

A shared human resource that can be assigned to stations requiring manual labor.

ParameterDescriptionDefault
CountNumber of operators in this pool.1
EfficiencyOperator efficiency as a percentage (1–100%). Scales effective cycle time.100%
SkillSkill label (e.g., “General”, “Welding”, “Assembly”). Stations can require specific skills.General

Assembly

Combines multiple input parts into one output. Waits until all required inputs are available.

ParameterDescriptionDefault
Cycle TimeSeconds to perform assembly.60s
Input PartsNumber of parts consumed per assembly (2–10). In simple mode, all inputs are generic.2
Per-Product InputsAdvanced mode: specify exactly which products and quantities are required.Simple mode

Disassembly

Breaks one input part into multiple output parts (reverse of assembly).

ParameterDescriptionDefault
Cycle TimeSeconds to disassemble.60s
Output PartsList of output products with quantities produced per disassembly.—

Palletize

Groups individual parts onto a pallet. The pallet continues downstream as a single entity.

ParameterDescriptionDefault
Default Pallet SizeNumber of items per pallet.10
Cycle TimeSeconds to palletize one batch.30s
Per-Product Pallet SizesOverride pallet size for specific product types.Uses default

Depalletize

Unloads all items from a pallet, releasing them individually downstream.

ParameterDescriptionDefault
Cycle Time per ItemSeconds to unload each item from the pallet.5s

Splitter

Routes incoming parts to one of 2–5 outputs based on a split strategy.

ParameterDescriptionDefault
OutputsNumber of output ports (2–5).2
Split Typeequal — round-robin. percentage — weighted probability. conditional — logic-based. product-based — route by product type.equal
Product RoutingFor product-based mode: map each product to a specific output port.—

Merge

Combines 2–5 input streams into a single output stream.

ParameterDescriptionDefault
InputsNumber of input ports (2–5).2
Merge Typefifo — first available. priority — by product priority. alternating — round-robin between inputs.fifo

Match Buffer

Holds parts until a complete set of required components is available, then releases them together. Useful for kitting or order-based assembly.

ParameterDescriptionDefault
CapacityMaximum parts the buffer can hold while waiting for matches.100
Match Keyorder — match by order ID. batch — match by batch ID.order
Required PartsList of product-quantity pairs that constitute a complete set.—
TimeoutOptional maximum wait time (seconds). Incomplete sets are released or scrapped after timeout.None

Products & Routing

Products are defined in the Orders > Product Manager. Each product has:

FieldDescription
NameDisplay name for the product.
RoutingOrdered list of station/node IDs the product visits. The simulation engine automatically creates arrival buffers for the first station in each routing.
Arrival RateSeconds between arrivals (used by sources in interval mode).
PriorityNumeric priority (lower = higher priority). Used by PRIORITY queue buffers and priority merge nodes.
Routing tip: A product’s routing defines the logical sequence of processing steps. The physical connections on the canvas define which nodes can transfer parts. Both must align for parts to flow correctly.

Connections

Connections define the material flow between nodes. Drag from one node’s output handle to another’s input handle in the Factory Builder.

PropertyDescription
SourceThe upstream node.
TargetThe downstream node.
ProbabilityOptional (0–1). When a node has multiple outgoing connections, probability values determine the routing split. If not set, parts are distributed equally. Must sum to 1 across all outputs from a node.

Resources

Resources represent shared capacity that stations compete for. Defined in the model and available in the Factory Builder.

FieldDescription
NameResource pool name.
Typeoperator, machine, or tool.
CapacityNumber of units available in the pool.
ShiftsOptional shift schedules (name, start hour, end hour, active days).
SkillsOptional skill tags. Stations can require specific skills.

Distribution Types

Cycle times, setup times, and other stochastic parameters can use any of these distributions:

DistributionParametersUse Case
constantvalueFixed, deterministic times (e.g., automated machines).
normalmean, stdSymmetric variation around a mean (e.g., manual assembly).
exponentialmeanMemoryless arrivals or service times (e.g., random arrivals).
triangularmin, mode, maxKnown best/worst/most-likely estimates.
uniformmin, maxEqual probability across a range.
lognormalmean, sigmaRight-skewed times (e.g., repair durations, complex manual tasks).
weibullshape, scaleReliability modeling (e.g., time-to-failure with aging effects).
empiricalvalues (array)Sampled from recorded data points.

OEE & KPIs

Overall Equipment Effectiveness (OEE)

OEE = Availability × Performance × Quality

ComponentFormulaMeaning
Availability (A)Average of each station’s uptime / total timeHow much time was the equipment available to run?
Performance (P)Average of each station’s (ideal CT × count) / (processing time + setup time)How fast did it run vs. ideal speed? Setup is counted as a speed loss.
Quality (Q)Line-level throughput / (throughput + scrapped)What fraction of output was good?
Note: Station-level OEE is computed in the engine (station.py:get_oee()). The overall OEE is aggregated in simulation.py:_calculate_oee() by averaging station A and P, and using line-level Q.

Throughput

Total parts completed during the simulation, broken down by product and by hour. The rate per hour is derived from total / duration.

Cycle Time

End-to-end time from when a part enters the system (source) to when it exits (sink). Reported as mean, standard deviation, min, and max, both overall and per product.

Work-In-Progress (WIP)

Total inventory in the system at any point. Reported as current total, per-buffer breakdown, and a time series chart showing WIP over the simulation.

Utilization States

Each station’s time is broken down into these states (percentages summing to 100%):

StateMeaning
BusyActively processing a part.
IdleAvailable but waiting for a part to arrive.
SetupPerforming a changeover/setup between batches or products.
BlockedFinished processing but cannot release the part because the downstream buffer is full.
StarvedReady to process but no parts available in the input buffer.
FailedEquipment is broken down (MTBF/MTTR).
Off-ShiftOutside the configured shift schedule.

Scenarios

Quick Scenarios

Pre-defined what-if templates that apply common changes with one click. Examples:

  • Double all buffer capacities
  • Reduce cycle times by 20%
  • Add a parallel station at the bottleneck
  • Increase arrival rate by 50%
  • Add a second shift

Each quick scenario runs the simulation with the override and compares results to the baseline.

Custom Scenarios

  1. Click New Custom Scenario on the Scenarios page.
  2. Select an entity type (station, buffer, source, conveyor, inspection, assembly, splitter, merge, disassembly, palletize, depalletize, match buffer).
  3. Pick the specific entity and the parameter to override.
  4. Set the new value. Repeat for multiple overrides.
  5. Name and save the scenario.
  6. Run it — results are stored alongside the scenario.

Comparing Scenarios

Select two or more scenarios from the list and click Compare. A side-by-side view shows OEE, throughput, cycle time, and WIP differences with percentage changes from the first (baseline) scenario.

Excel Import / Export

Importing

Go to Data Sync and choose Import Excel or Import CSV.

  • Select the file and sheet (for Excel).
  • Map columns to FactorySim parameters.
  • Preview the import and confirm.
  • Errors and warnings are shown per row.

Exporting

Use Dashboard > Export Report to save simulation results as CSV or JSON. The CSV includes:

  • Summary section: OEE, throughput, cycle time, WIP
  • Station utilization breakdown
  • Hourly throughput time series

Round-Trip Workflow

  1. Export your current model parameters to Excel.
  2. Edit values in the spreadsheet (batch update cycle times, capacities, etc.).
  3. Re-import the modified spreadsheet to update the model.
